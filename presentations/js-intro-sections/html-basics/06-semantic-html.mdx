## Semantic HTML

<CodeDisplay>
  <File name="index.html">
```html
<!DOCTYPE html>
<html>
<head>
    <title>Semantic HTML Example</title>
</head>
<body>
    <header>
        <h1>Habit Tracker</h1>
        <nav>
            <ul>
                <li><a href="#dashboard">Dashboard</a></li>
                <li><a href="#habits">My Habits</a></li>
                <li><a href="#stats">Statistics</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <article class="habit-summary">
            <header>
                <h2>Morning Exercise</h2>
                <time datetime="2024-01-15">Started Jan 15, 2024</time>
            </header>
            <section>
                <p>Current streak: <strong>15 days</strong></p>
                <progress value="15" max="30">50%</progress>
            </section>
            <footer>
                <button>Mark Complete</button>
                <button>View Details</button>
            </footer>
        </article>
        
        <aside>
            <h3>Quick Stats</h3>
            <dl>
                <dt>Total Habits</dt>
                <dd>5</dd>
                <dt>Longest Streak</dt>
                <dd>30 days</dd>
            </dl>
        </aside>
    </main>
    
    <footer>
        <p>&copy; 2024 Habit Tracker App</p>
    </footer>
</body>
</html>
```
  </File>
</CodeDisplay>
<Notes>
This example showcases semantic HTML in action, demonstrating how proper element selection creates meaningful document structure. Every element tells a story about its content's purpose and relationships.

The page structure uses landmark elements that assistive technologies recognize. The header element at the page level contains site-wide navigation and branding. Screen readers can jump directly to these landmarks, making navigation efficient. The nav element explicitly marks navigation, and using a list for navigation items indicates their relationship.

The main element identifies the primary content, excluding repeated elements like headers and footers. Within main, we use article for the habit summary because it's self-contained content. Notice how article has its own header - semantic elements can nest, creating hierarchical relationships.

The footer demonstrates how metadata can be presented semantically. The time element provides machine-readable dates while displaying user-friendly text. Small print (small element) indicates fine print or side comments. This semantic richness helps search engines understand your content, improves accessibility, and makes your HTML self-documenting. When you read semantic HTML, you understand the content's purpose without seeing the rendered page.
</Notes>
